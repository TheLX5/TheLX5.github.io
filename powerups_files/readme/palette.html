Uploading your own palette.


Like Manual Player Palette Uploader patch, the powerup pack lets you upload your own Player palette any time you want by writing the location of your palette table to a 24-bit pointer in RAM.
The RAM pointer can be used with the !pal_pointer define and to activate the effect, you also need to set !pal_bypass to a non-zero value in order to upload your palette. Note that !pal_bypass is cleared every frame and the original palette will be uploaded when it is zero.

A small example of this feature:

  LDA #$01
  STA !pal_bypass
  LDA #YourPalette
  STA !pal_pointer
  LDA.b #YourPalette/$100
  STA !pal_pointer+1
  LDA.b #YourPalette/$10000
  STA !pal_pointer+2

  YourPalette:
  dw $4F3F,$581D,$1140,$3FE0,$3C07,$7CAE,$7DB3,$2F00,$165F,$03FF

 If you prefer using .mw3 files instead of raw SNES RGB palette data, use the %insert_palette(filename) macro, which only inserts the relevant data (player palettes). The macro is also included in UberASM as %insert_palette(path).
