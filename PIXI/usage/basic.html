Workspace & files [Link]

PIXI comes with the ideal workspace where it performs the best; a isolated folder without other tools (other than CFG Editor) inside of it.

PIXI's default file structure.

The files inside the .zip file have their very own purpose and it's not recommended to remove or rename them.

Summary of PIXI's files and their functions

Folder/File Description
asm This folder contains the ASM source for everything PIXI requires to work in a ROM.
cluster Default directory for Cluster Sprites. This is where those sprite types go.
extended Default directory for Extended Sprites. This is where those sprite types go.
generators Default directory for Generator Sprites. This is where those sprite types go.
misc_sprites Default directory for Misc Sprites. Inside of this folder are more folders for different, albeit minor, sprite types that PIXI can insert.
routines Default directory for Shared Routines. This is where every shared routine must go in order to be used by PIXI and its sprites.
shooters Default directory for Shooter Sprites. This is where those sprite types go.
sprites Default directory for Custom Sprites. This is where those sprite types go.
asar.dll Asar in .dll form.
CFG Editor.exe Old CFG Editor. Lets you modify certain behaviors of the Custom Sprites you insert. You're required to insert sprites again after modifying the .cfg or .json files.
changelog.txt Old changelog file.
Newtonsoft.Json.dll JSON parser for PIXI.
pixi.exe Executable file for PIXI. Inserts sprites mentioned in the list file.
readme.txt Old readme file with instructions.
removedResources.txt List of every resource removed from PIXI in v1.2.10.

Sprite types [Link]

PIXI lets you insert several sprite types into your ROM, which should cover every sprite type in the original game.

PIXI's supported sprite types

Sprite type Examples
Regular/Custom Koopas, Goombas, Rexes and Blurps
Shooter Bullet Bill cannons and Torpedo Ted launchers
Generator Fish generator.
Extended Baseballs from Pitchin' Chucks and Player's fireballs.
Cluster Boos in a circle & Sumo Bros' flames from a lightning.
Bounce Appear above of a Question Mark Block that the player just hit.
Smoke Small particles when the player quickly turns around.
Minor Extended Exploding particles from a broken Throw block or Turn block.
Score Text that appears upon defeating any enemy.
Spinning Coin Coin that appears when hitting a Question Mark Block from below which contains one or more coins.

PIXI sprite list [Link]

The default name for a list file is list.txt and should be in the same folder as your ROM file.

To insert sprites with PIXI you must create a list file containing a list of sprite you want to insert into your ROM. This list file has a very specific format that the end user has to follow to avoid errors or ROM corruptions.

List format:


    <sprite ID/number> SPACE <path to sprite file>

Example:


    00 Blue.cfg
    01 Red.json
    02 Koopas/Green.json
    03 Koopas/Yellow.json
Do note that every sprite ID should be in hex and two digits, meaning that you're only able to insert sprites with IDs within the range of 00-FF (if the sprite type allows this). The hex number should not have any prefix to indicate that it's a hex number (no 0x, $ or any other notation).

You can also have comments in your list file, everything after a semicolon (;) will be treated as a comment:


    00 Blue.cfg             ; insert Blue.cfg
    01 Red.json             ; insert Red.json with an ID of 01
    02 Koopas/Green.json    ; insert Green.json, which is inside of the Koopas folder
    03 Koopas/Yellow.json
    ; comments at the start of a line are valid as well

Every sprite type PIXI supports has a limit on how many sprites you can insert with the tool, they also require a header within the list file to let the tool know that the following IDs/sprites are another kind of sprite.

Sprites that aren't regular, shooters or generators usually require pointing to .asm files instead of .cfg or .json files.


    SPRITE:                             ; you can ommit this line if needed
    00 Green.json                       ; Green
    1F SnakeBlock.cfg                   ; Custom Snake block
    BF Powerups/PoisonMushroom.json     ; Poison Mushroom
    C3 HomingBillLauncher.cfg           ; Shooter that spawns homing bullet bills
    D1 LowGravity.json                  ; Generator for low gravity

    ; We're done with other sprite types, anything below will be treated as a extended sprite
    EXTENDED:                           
    08 LotusBlueFire.asm                ; Blue fire for Lotus Plants

    ; Now we're inserting cluster sprites, not extended sprites
    CLUSTER:
    01 fish.asm                         ; Layer 3 fish but in cluster sprite form!

    ; ... and now we're inserting bounce sprites! Probably you get the idea at this point
    BOUNCE:
    07 red_note_block.asm               ; Recreation of SMB3's red note block

    SMOKE:
    00 gray_smoke.asm                   ; Regular smoke, but uses palette 9

    MINOREXTENDED:
    1D IceBlockShatter.asm              ; Particles for a weird ice block

    SCORE:
    02 7up.asm                          ; Because 5up wasn't enough

    SPINNINGCOIN:
    0E BlueCoin.asm                     ; Blue coin from blocks

Inserting sprites [Link]

PIXI is a tool with a command line interface, meaning that there's no fancy GUI to help you insert sprites into your ROM or create a list file. Despite being like this, the tool is fairly straight forward to use.

By default PIXI will search a list file named list.txt in the same folder as the ROM file.

Step 1

Open PIXI.exe by double clicking the executable file or opening it directly from the command line window.

PIXI's command line interface

Step 2

Write the path to your ROM file in the command line window or drag the file into the window.

ROM name provided to PIXI

Step 3

Wait until PIXI is done inserting sprites. It may take a while to insert each sprite, so be patient and don't close the window if there's nothing going on.

Sprite insertion done without issues.
