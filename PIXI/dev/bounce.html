PIXI defined RAM [Link]

Define RAM address
LoROM		 RAM address
SA-1		 Description
!bounce_table_2 [Link] !bounce_map16_high $1968 $7968

Extra table provided by PIXI.

Holds the high byte of the Map16 tile number the bounce sprite will revert to.

WARNING: If 255 sprites per level are disabled, the RAM addresses mentioned here will be remapped to a long address. Please refer to the appropriate section for more information.

Original game RAM [Link]

Define RAM address
LoROM		 RAM address
SA-1		 Description
!bounce_num [Link] $1699 $7699

Bounce sprite type number.
!bounce_init [Link] $169D $769D

Bounce sprite initialization flag table. #$00 = bounce sprite in init routine; #$01 = bounce sprite in main routine. Used for several things, such as generating tile 152 (invisible solid) once only.
!bounce_y_low [Link] $16A1 $76A1

Bounce sprite Y position, low byte.
!bounce_x_low [Link] $16A5 $76A5

Bounce sprite X position, low byte.
!bounce_y_high [Link] $16A9 $76A9

Bounce sprite Y position, high byte.
!bounce_x_high [Link] $16AD $76AD

Bounce sprite X position, high byte.
!bounce_y_speed [Link] $16B1 $76B1

Bounce sprite Y speed.
!bounce_x_speed [Link] $16B5 $76B5

Bounce sprite X speed.
!bounce_x_bits [Link] $16B9 $76B9

Accumulating fraction bits for bounce sprite X speed.
!bounce_y_bits [Link] $16BD $76BD

Accumulating fraction bits for bounce sprite Y speed.
!bounce_map16_tile [Link] !bounce_map16_low $16C1 $76C1

Bounce sprite turns into Map16 tile. This uses the same values as $7E:009C.

This table is also used as the low byte of the Map16 tile number the bounce sprite will revert to if the sprite doesn't make use of SMW's $00BEB0.
!bounce_timer [Link] $16C5 $76C5

Bounce sprite timer - amount of frames until bounce sprite disappears. Turn blocks are set to spinning mode when this timer runs out.
!bounce_table [Link] !bounce_table_1 $16C9 $76C9

Block bounce sprite table.

Format: L-----DD

    L is which layer it is on. Clear means it's on layer 1, set means it's on layer 2 (or layer 3 if applicable).
    DD is the direction it is moving in. 00 = up; 01 = right; 10 = left; 11 = down.
!bounce_properties [Link] $1901 $7901 YXPPCCCT data of bounce sprite that is being shown.
